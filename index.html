Here is a link to the first site I built (at 12 days into learning this people, cut me a break). I was soo proud of it at the time, but I have a feeling I'm going to be gravely embarassed by it in the future ha.

I'm working with my friend to build a site together and coordinate that build with GitHub, so I thought why not blog my way through this process as well. Especially since GitHub lets me save each iteration of my process, I think it could be really cool to look back on - like a diary of code.

I want the form, structure & content on this platform to evolove as my coding knowledge does. I also want provide a candid account of what the path from zero computer knowledge to fullstack developer looks like (- confidence, bitches -) but without leaving out any of the sticky or hard stuff to try to make myself sound a particular breed of smart or knowledgable.

Goals for next time: map out how these blog posts should flow and be archived so they're not just sitting here in a blob, but can be actually organized by date and key accomplishments.

    These are things I learned today:
  • you can't link html to local image files
  • background images aren't easy - perhaps even possible? - to put in a line
  • css can turn your normal elements in html into table elements without you having to set them up that way in html first! so cool!
  • if I use 'text-align: center' in css on a container, the images (or text) inside that container becomes centered within that on the page

This is going to be a long road, and I'm so excited to be here at the start! Wish me luck!!
